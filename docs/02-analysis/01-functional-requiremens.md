### 1. Регистрация и управление учетной записью:

1.1. **Регистрация на платформе:**

- **Источник требования:** Все пользователи.
- **Описание требования:** Реализовать функционал регистрации на платформе с обязательными полями для заполнения и
  подтверждением через электронную почту. Поля для заполнения включают:
    - Электронная почта (обязательное поле)
    - Пароль (обязательное поле)
    - Псевдоним(Nickname)
- **Приоритет требования:** Высокий.

1.2. **Вход в систему:**

- **Источник требования:** Все пользователи.
- **Описание требования:** Обеспечить возможность аутентификации пользователей по электронной почте и паролю.
- **Приоритет требования:** Высокий.

### 2. Управление конкурсами:

2.1. **Создание/Редактирование/Удаление конкурсов:**

- **Источник требования:** Организаторы конкурсов
- **Описание требования:** Реализовать функционал создания/редактирования/удаления конкурсов. Поля для создания
  конкурса включают:
    - Тариф
    - Название конкурса
    - Правила конкурса
    - Дата начала конкурса
    - Критерии приемки работ
    - Аватар конкурса
    - Количество отборочных этапов
    - Количество участников для этапа плей-офф
    - Имена пользователей модераторов
    - Имена пользователей судейской коллегии
- **Приоритет требования:** Высокий.

2.2. **Поиск пользователей:**

- **Источник требования:** Организаторы конкурсов.
- **Описание требования:** Обеспечить возможность поиска пользователей по псевдониму для назначения ролей при
  создании/редактировании конкурса.
- **Приоритет требования:** Высокий.

2.3. **Поиск конкурсов:**

- **Источник требования:** Все пользователи
- **Описание требования:** Обеспечить возможность поиска конкурсов с возможностью фильтрации и сортировки по
  следующим полям:
    - Название конкурса
    - Статус
    - Наличие призового фонда
    - Организатор
- **Приоритет требования:** Средний.

2.4 **Бэкап данных конкурса**

- **Источник требования:** Организаторы
- **Описание требования:** Обеспечить автоматическое сохранение/очистку данных конкурса на время определенное в тарифе
- **Приоритет требования:** Высокий.

2.5 **Сохранение/Развертывание данных конкурса**

- **Источник требования:** Организаторы
- **Описание требования:** Обеспечить возможность сохранения архива с бэкапом конкурса и возможность восстановления из
  него данных конкурса в панели организатора.
- Данные которые должны сохраняться в архиве:
    - Название конкурса
    - Аватар конкурса
    - Судейская коллегия и модераторы
    - Все работы участников
    - Данные о завершенных этапах (правила и сроки проведения, выставленные оценки)
    - Данные о не завершенных этапах (кроме сроков завершения)
    - Цифровая подпись
        - Формируется по алгоритму скрытому от пользователя на основе информации о конкурсе
        - Хранится в зашифрованном виде
        - Необходима для восстановления конкурса на платформе из бэкапа, для того что бы исключить скрытую подтасовку
          результатов, при несовпадении подписи, восстановить бэкап будет невозможно
- **Приоритет требования:** Высокий.

2.6 **Выставление/Просмотр оценок**

- **Источник требования:** Организаторы/Судейская коллегия
- **Описание требования:** Обеспечить возможность оценки работ сданных на конкурсы по критериям заданным
  организатором для данного этапа. Возможные поля для оценки работ:
    - Название критерия оценки
    - Тип оценки
        - true/false
        - цифровая шкала
        - текстовый комментарий
- **Приоритет требования:** Высокий.

2.7. **Создание/Редактирование/Удаление этапов конукрса:**

- **Источник требования:** Организаторы конкурсов
- **Описание требования:** Реализовать функционал создания/редактирования/удаления этапа конкурса из заранее
  предложенных шаблонов. Должно быть доступно 2 варианта этапов - отборочные этапы и плей-офф.
- Отборочные этапы:
    - Все участники сдают свои работы в рамках требований описанных организатором
    - Судейская коллегия оценивает работы по критериям заданным организатором для этого этапа
    - После завершения оценки система автоматически определяет участников прошедших в следующий этап по порогу
      заданному организатором:
        - По необходимому количеству участников для формирования следующего этапа в случае если следующий этап плей-офф
        - По пороговому значению набранных баллов оценки (при наличии критериев оценки поддающихся подсчету)
    - При необходимости организатор в ручном режиме может определить участников прошедших в следующий этап:
        - При возникновении спорных оценок
        - При отсутствии критериев поддающихся подсчету для автоматического определения участников прошедших в следующий
          этап
    - Завершить этап конкурса можно только если всем работам выставлены оценки минимальным количеством членов
      судейской коллегии определенных в правилах этапа
    - После завершения этапа сдача работ на следующий этап возможна только для участников прошедших в следующий этап
      по результатам оценки
- Плей-офф:
    - Все участники делятся на пары в ручном или автоматическом/полуавтоматическом режиме
    - Для этапов плей-офф обязательным является наличие критериев оценки поддающихся подсчету
    - Судейская коллегия оценивает работы по критериям заданным организатором для этого этапа
    - После завершения оценки система автоматически определяет участников прошедших в следующий этап.
    - В спорных случаях должна быть возможность определения победителя в рамках пары:
        - Добавление дополнительного члена судейской коллегии для данной пары
        - Решение на усмотрение организатора
        - Зрительское голосование
- Наличие плей-офф этапов не является обязательным
- **Приоритет требования:** Высокий.

### 3. Управление композициями:

3.1 **Загрузка/Редактирование/Удаление композиций**

- **Источник требования:** Организаторы/Участники конкурсов
- **Описание требования:** Обеспечить возможность загрузки файлов в форматах mp3/wav/flac в зависимости от тарифа
  выбранного организатором, а так же их редактирование и удаление. Возможность редактировать и удалять работу
  должна быть только до начала её оценки, либо до завершения приема работа на этап.
- Поля для загрузки/редактирования работы:
    - Название композиции *
    - Год выпуска
    - Композитор
    - Автор стихов
    - Текст композиции
    - Доступ для скачивания *
    - Ссылки на стриминговые сервисы
    - Жанр *
    - Обложка
- **Приоритет требования:** Высокий.
